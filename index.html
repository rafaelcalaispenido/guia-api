Segurança

Autenticação

Todos os métodos da API exigem autenticação prévia por meio da sua API Key, enviada em toda requisição.

Chave API

  1. Para ter acesso à sua chave API, acesse o eNotas e realize login em app.enotas.com.br.
  2. Clique no nome que aparece no canto superior direito.
  3. Selecione Perfil e depois Acessar configurações para nerds.
  4. Irá aparecer a opção Clique aqui para visualizar sua API Key — copie a API Key exibida.

URL base

Base URL
https://app.enotas.com.br/api
Se você utilizar qualquer outra URL base, receberá 404 – Not Found ao enviar a requisição.

Header de autenticação

Inclua sua API Key no header Authorization usando Basic Authentication:

Header
Authorization: Basic {sua-API-Key-aqui}
HeaderValorObrigatório
AuthorizationBasic {sua-API-Key}Sim
Content-Typeapplication/jsonSim (em POST)
Acceptapplication/jsonRecomendado
Erro comum: espaços em branco ou quebras de linha acidentais no valor da API Key colada no header. Sempre confira se não sobrou nenhum caractere extra antes de enviar.

Erro AUT002 — API Key inválida

Se houver qualquer incoerência na API Key informada, a API retorna:

Resposta 401
{
  "codigo": "AUT002",
  "mensagem": "Acesso negado. APIKey invalida, por favor verifique se setou o header do request corretamente com uma APIKey valida, por exemplo: Authorization: Basic {your-APIKey-here}."
}
  • Revise se a API Key foi copiada por completo, sem cortar caracteres.
  • Remova espaços, quebras de linha ou caracteres invisíveis colados junto com a chave.
  • Confirme que o header está exatamente como Authorization: Basic {sua-API-Key}.

Endpoints

Fluxo Principal (CRUD)

Os seis endpoints centrais para integrar clientes, produtos e vendas, do cadastro até a emissão e o cancelamento da nota fiscal.

POST/clientes

Cria ou atualiza um cliente (tomador da nota). Se o tomador for Pessoa Jurídica, informe o CNPJ — ao enviar a nota para emissão, o sistema busca automaticamente os dados na Receita Federal.

Request Body
{
  "cpfCnpj": "22255588859",
  "nome": "Cliente Teste API",
  "email": "teste@enotas.com.br",
  "telefone": "31231231231",
  "quandoEmitirNFe": 1,
  "enviarNfePorEmail": true,
  "ignorarAtualizacaoAutomatica": true,
  "endereco": {
    "pais": "Brasil",
    "uf": "MG",
    "codigoIbgeUf": 31,
    "cidade": "Belo Horizonte",
    "codigoIbgeCidade": 3106200,
    "logradouro": "Rua teste",
    "numero": "427",
    "complemento": null,
    "bairro": "Centro",
    "cep": "30130110"
  }
}
Campos importantes
nome / emailUm dos dois é obrigatório.
cpfCnpjOpcional. Limite: CPF 11 / CNPJ 14 caracteres.
enderecoObrigatório ou não conforme regra da prefeitura do tomador.
endereco.logradouroLimite de 1.000 caracteres.
endereco.cepLimite de 8 caracteres.
quandoEmitirNFeVer enumeração na seção 4.
Resposta 200
{
  "clienteId": "7a1a8765-08de-9509-49ba-3d3094d264b4"
}

GET/vendas/getFilterBy

Consulta paginada de vendas, com filtros no estilo OData sobre a venda, sobre a nota fiscal e sobre tags.

ParâmetroObrigatórioDescrição
pageNumberSimNúmero da página, começando em 0.
pageSizeSimQuantidade de itens por página.
filterNãoFiltros sobre a venda: data, dataPagamento, situacao, tags/id, cliente/id, cliente/nome, cliente/cpfCnpj, produto/id.
nfeFilterNãoFiltros sobre a nota: nfe/situacao, nfe/dataCompetencia, nfe/prefeituraNumero, nfe/rpsNumero.
tagFilterNãoFiltro por tag numérica. Ex.: tags/id eq 25358.
orderByNãodata, situacao, valorTotal, cliente/nome, produto/nome, nfe/situacao. Ex.: data desc.

Operadores OData suportados

eq
igual
contains
contém

maior ou igual

menor ou igual

Exemplos de URL

Consulta por CPF/CNPJ do cliente
GET /clientes/getFilterBy?pageNumber=0&pageSize=50&filter=(contains(cpfCnpj,'22255588859'))
Consulta por número da nota fiscal (faixa)
GET /vendas/getFilterBy?pageNumber=0&pageSize=10&filter=nfe/prefeituraNumero ge '1000' and nfe/prefeituraNumero le '2000'&orderBy=data desc
Consulta venda por ID
GET /vendas/{vendaId}

POST/vendas/{vendaId}/cancelar

Cancela uma venda e solicita o cancelamento da nota fiscal associada junto ao órgão emissor (prefeitura ou Sefaz).

Path Parameter
POST /vendas/79f35850-08de-af9f-4f32-f28a4bff2bf0/cancelar
Resposta 200
{
  "vendaId": "79f35850-08de-af9f-4f32-f28a4bff2bf0"
}
Atenção: algumas prefeituras não suportam cancelamento ou têm prazo curto para isso. A NF-e de Produto tem prazo de 24h para ser cancelada após a emissão. Cancelar a venda não garante que a nota também será cancelada — acompanhe nfe.situacao para confirmar.

Enumerações e Configurações

Valores inteiros usados pelos campos da API para representar estados e opções de configuração.

Situação da Venda campo situacao

ValorNomeDescrição
0EmAbertoO pagamento da venda ainda não foi confirmado. Geralmente ocorre em vendas geradas por boleto.
2EmGarantiaPagamento confirmado; se houver período de garantia, a venda aguarda o vencimento dele.
3CompletaPagamento confirmado e garantia (se houver) já vencida. Sem garantia, completa automaticamente após o pagamento.
5CanceladaVenda cancelada (reembolso, chargeback ou cancelamento manual). Não implica que a nota fiscal também foi cancelada.

Situação da NFe campo nfe.situacao

ValorNomeDescrição
0PendenteVenda em garantia ou em aberto aguardando pagamento. Ao concluir, a nota é emitida automaticamente.
1EmProcessoDeEmissaoEm processamento na prefeitura/Sefaz. Vendas de afiliado/marketplace podem levar até 30 dias; vendas normais, poucos minutos.
2EmitidaNota já emitida na prefeitura ou Sefaz.
3FalhaAoEmitirRetornou com erro. É preciso corrigir o dado e reenviar.
4EmProcessoDeCancelamentoA venda foi cancelada e o cancelamento da nota já emitida está em andamento.
5CanceladaCancelamento da nota confirmado na prefeitura ou Sefaz.
6FalhaAoCancelarA prefeitura/Sefaz recusou a tentativa de cancelamento.
7NaoEmitirNão será emitida automaticamente — requer reenvio manual pelo painel.
8EnviadoPorEmailPDF enviado ao tomador (requer e-mail informado e enviarNFeCliente: true).

Boas práticas

Dicas de Implementação

Comportamentos automáticos, limites de campos e particularidades por prefeitura que costumam gerar dúvida durante a integração.

Quebra de linha em observações

Para quebrar linha no campo observacoes, inclua \r\n ou \n no JSON de envio da venda. Algumas prefeituras não suportam quebra de linha — por exemplo, Goiânia/GO. Além disso, não é possível quebrar linha em nota fiscal de Produto (NF-e).

Prazo de cancelamento de NF-e de Produto

A NF-e de Produto tem prazo de 24 horas após a emissão para ser cancelada via POST /vendas/{vendaId}/cancelar. Passado esse prazo, o cancelamento pela API deixa de ser aceito pelo Sefaz.

Busca automática de dados na Receita Federal

Se o tomador for Pessoa Jurídica e você informar apenas o cpfCnpj, a API busca automaticamente razão social e endereço na Receita Federal ao enviar a nota para emissão — reduzindo a necessidade de coletar todos os dados manualmente.

Cálculo automático de ISS

Se o campo aliquotaIss não for informado no produto ou na venda, o sistema calcula automaticamente o ISS conforme a alíquota vigente no município de prestação do serviço.

Além da quebra de linha em Goiânia/GO, algumas prefeituras não suportam ou limitam o cancelamento de NFS-e após certo prazo. Como cada município tem regras próprias definidas por sua legislação, sempre valide o comportamento específico com o suporte eNotas antes de assumir que um recurso está disponível para a cidade do seu cliente.

Troubleshooting

Tratamento de Erros

Todo código de resposta segue o padrão HTTP: 2xx indica sucesso, 4xx indica erro do lado do cliente e 5xx, instabilidade da eNotas.

StatusSignificadoComo agir
200SucessoMétodo executado com sucesso.
400Falta informação obrigatóriaAlgum campo obrigatório não foi enviado no corpo da requisição. Revise contra a documentação do endpoint.
401Falha ao autenticarAPI Key ausente, incorreta ou com espaço/quebra de linha. Veja o erro AUT002 na seção Autenticação.
403Não autorizadoA conta não tem permissão para o recurso solicitado — confira o plano contratado (Business/Enterprise).
404Not FoundURL inválida ou recurso (cliente/produto/venda) inexistente com o ID informado.
415Informação inválida no HeaderConfira se Content-Type está definido como application/json.
500Erro internoInstabilidade momentânea na eNotas. Aguarde alguns segundos e tente novamente; persistindo, acione o suporte.

Código específico: AUT002

401 — AUT002
{
  "codigo": "AUT002",
  "mensagem": "Acesso negado. APIKey invalida, por favor verifique se setou o header do request corretamente com uma APIKey valida."
}

Checklist de debug

  • Confira se a URL usa exatamente https://app.enotas.com.br/api como base.
  • Valide o JSON enviado em um linter antes de disparar a requisição (chaves e vírgulas são a causa mais comum de 400).
  • Verifique se todos os campos obrigatórios do endpoint (ver seção 3) foram preenchidos.
  • Confirme que o header Content-Type: application/json está presente em toda chamada POST.
  • Em erros 404 de consulta, confirme se o ID pertence à mesma conta/API Key usada na chamada.
  • Em erros 500 recorrentes, registre o horário e o corpo da requisição antes de abrir um chamado com o suporte.

Visão geral

Fluxo de Integração

Sequência típica de chamadas para levar uma venda do cadastro até a nota fiscal emitida — e os pontos de decisão ao longo do caminho.

1
Criar Cliente POST /clientes
Cadastre o tomador da nota. Pode ser feito isoladamente ou embutido diretamente na criação da venda.
2
Criar Produto POST /produtos
Cadastre o item vendido, com valor e, se aplicável, dias de garantia. Também pode ser embutido na venda.
3
Criar Venda POST /vendas
Vincula cliente e produto, define valores, meio de pagamento e a regra de emissão da nota.
?
Decisão: quando emitir a NFe
Definido pelo campo quandoEmitirNFe no momento da criação da venda.
0 · Na venda → emite na hora 1 · Após garantia → emite ao vencer 2 · Não emitir → requer chamada manual 3 · Em aberto → aguarda outra condição
4
Emitir NFe POST /vendas/emitirNFe
Chamada automática pela eNotas (cenários 0/1) ou manual pelo seu sistema (cenários 2/3, ou reenvio após falha).
